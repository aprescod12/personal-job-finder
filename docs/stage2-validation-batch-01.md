# Stage 2 Unseen Validation Batch 01

Research date: **2026-07-17**

This holdout batch tests matcher version `2.3-controlled-semantic` on ten opportunities that were not used to design the weights, role pathways, vocabulary, or semantic concept families.

## Why the result is hidden

A validation job remains in blind-review mode until a human judgment is saved. Before that point, the dashboard, job-detail page, and match-analysis page do not render:

- match score
- fit classification
- opportunity lane
- evidence coverage
- direct, related, semantic, or missing-evidence counts
- blocker conclusions generated by the matcher

The structured posting requirements remain visible because they are the evidence the human reviewer should evaluate.

## Load the batch

Preview without changing the database:

```bash
python manage.py load_stage2_validation_batch --dry-run
```

Load the ten jobs:

```bash
python manage.py load_stage2_validation_batch
```

The command is idempotent. It never creates a human calibration and never overwrites a manually entered job that uses the same URL. `--refresh` restores curated fields only on records originally managed by this validation batch.

## Review protocol

Use this dashboard view:

```text
SOURCE: Validation holdout
CALIBRATION: Not yet reviewed
SORT: Company A–Z
```

For each job:

1. Open the original source and verify the opportunity details.
2. Review the structured role, industry, skills, education, experience, and blockers.
3. Decide whether the opportunity is a Strong, Possible, Weak, or Not Eligible fit.
4. Choose Priority, Adjacent, Outside Priority, or Unsure.
5. Write the strongest reason for the judgment.
6. Select **Save and Reveal Result**.
7. Observe the comparison, but do not alter matcher rules until all ten jobs are reviewed.

## Evaluation rules

After all ten reviews, filter the calibration report to **Unseen validation holdout**.

Focus on:

- fit agreement
- opportunity-lane agreement
- false strong or good matches
- false disqualifications
- experience-gap handling
- authorization handling
- accepted and rejected semantic matches

Do not optimize the matcher solely to make every holdout result agree. A disagreement should lead to a rule change only when the reason generalizes beyond one posting.

## Source limitation

Job postings and program pages can change or close. The importer stores dated, concise requirement summaries rather than complete copyrighted descriptions. Always verify the live source before applying.
